Fundamentals First

Once you have the fundamentals, progress accelerates.

Fundamentals before Frameworks

Rather than jumping immediately into heavy tooling, I wanted to be comfortable manipulating the DOM, debugging JavaScript errors, managing state manually, and understanding how CSS behaves across screen sizes.

The real win wasn’t just getting things to work it is understanding why they work, where they belong, and how to adapt when environments change.

Why Fundamentals Matter

This paid off quickly. When issues arose, I could trace the problem, inspect it in dev tools, and fix it with confidence.

A portfolio doesn’t need to be perfect, it needs to be honest.

This is the growth that compounds.
