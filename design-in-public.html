Designing in Public

When I started working on Bells & Whistles, the goal wasn’t to build a “perfect” portfolio. It was to build something that could evolve.

From Static to Intentional

The first version of my portfolio was a single static page. It existed, but it didn’t communicate how I think or work.

By treating the site as a living product, I gave myself permission to iterate instead of restart.

Why Visibility Matters

Designing in public forced me to make decisions visible, and that visibility, sharpened my thinking.

My portfolio doesn’t need to be perfect, it needs to be honest.

Each iteration added clarity, not polish for polish’s sake.
